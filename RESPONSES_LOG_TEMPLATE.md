# Лог откликов — шаблон

По заданию требуется лог 60 откликов. Этот файл содержит структуру и шаблоны сообщений.

## Таблица откликов

Используйте файл `responses_log_template.csv` и заполняйте по факту отправки.

Рекомендуемые поля:

- `number` — номер отклика;
- `date` — дата;
- `platform` — площадка;
- `company_or_contact` — компания / контакт;
- `vacancy_or_request` — вакансия / запрос;
- `focus` — фокус A или B;
- `message` — текст отклика;
- `link` — ссылка на вакансию / запрос;
- `status` — sent / replied / rejected / interview / no_response;
- `screenshot_file` — имя файла со скрином;
- `notes` — заметки.

## Фокус A: AI triage + черновики ответов

Здравствуйте! Могу внедрить AI-сервис для первичной обработки обращений: классификация + черновик ответа + эскалация сложных случаев.

Сделаю программный интерфейс, хранение истории и базовые ограничения по безопасности / лимитам. Уточните, какие каналы обращений и какие категории вам нужны?

## Фокус B: деплой и production-подход

Здравствуйте! Могу развернуть ИИ-сервис: стабильная работа, секреты в переменных окружения, логирование, журнал запросов, сценарий эскалации при сбоях.

Покажу демо и отдам инструкцию, чтобы решение поддерживалось. Можете описать текущий процесс обработки обращений и объёмы?

## Что прикладывать как доказательство

- скрин отправленного отклика;
- ссылка на вакансию / запрос;
- дата отправки;
- статус;
- при ответе — скрин или краткая заметка.
